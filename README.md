This project demonstrates negative and edge-case API testing using Postman.
It focuses on validating system behavior under invalid input, unauthorized access,
and non-existing resources.

## Tools
- Postman
- JavaScript (Postman test scripts)
- Fake Store API

## Covered Scenarios
- Login failure with invalid credentials
- Unauthorized access to protected endpoints
- Handling of non-existing user resources
- Validation of HTTP status codes and error behavior

## Execution
1. Import the collection into Postman
2. Import the environment
3. Select the Fake Store environment
4. Run tests using Collection Runner

## Status
All negative test cases pass successfully.
